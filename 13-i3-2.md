# 13 — i3 على Void Linux (التحويل + التشغيل مع Sway)

---

## الفرق الجوهري في جملة واحدة

| | Sway | i3 |
|---|---|---|
| **Display Server** | Wayland | X11 / Xorg |
| **الـ Config** | ~99% متطابق | ~99% متطابق |
| **Compositor** | مدمج | محتاج `picom` منفصل |
| **Screen Sharing** | عبر Portal + PipeWire | مباشر (أسهل) |

  i3lock \

  xclip \
  arandr \

## تشغيل i3 من TTY

أنشئ `~/.xinitrc`:

```bash
#!/bin/sh

# ⚠️ dbus لازم أول حاجة — بدونه PipeWire والـ Keyring مش هيشتغلوا
eval $(dbus-launch --sh-syntax)
export DBUS_SESSION_BUS_ADDRESS

# متغيرات X11
export XDG_SESSION_TYPE=x11
export XDG_SESSION_DESKTOP=i3
export XDG_CURRENT_DESKTOP=i3

# Keyring (بعد dbus)
eval $(gnome-keyring-daemon --start --components=gpg,pkcs11,secrets,ssh \
    | sed 's/^\(.*\)/export \1/g')
export SSH_AUTH_SOCK

# PipeWire (بعد dbus)
pipewire &

exec i3
```

```bash
chmod +x ~/.xinitrc
```

لتشغيله:
```bash
startx
```

> ⚠️ **الخطأ الشائع:** لو شغّلت PipeWire قبل dbus أو في كونفيج i3 بدل الـ xinitrc، الصوت مش هيشتغل. الترتيب الصح: `dbus → keyring → pipewire → i3`.

## تشغيل تلقائي من TTY1 (i3) أو TTY2 (Sway)

في `~/.profile`:

```bash
# TTY1 → Sway
if [ -z "$WAYLAND_DISPLAY" ] && [ "$(tty)" = "/dev/tty1" ]; then
    exec ~/.local/bin/start-sway
fi

# TTY2 → i3
if [ -z "$DISPLAY" ] && [ "$(tty)" = "/dev/tty2" ]; then
    exec startx
fi
```

> الآن: `tty1` = Sway / `tty2` = i3. للتنقل بينهم: `Ctrl+Alt+F1` و `Ctrl+Alt+F2`.

## كونفيج i3 (نسخة كاملة عشان تبدأ بيها)

```bash
mkdir -p ~/.config/i3
cp /etc/i3/config ~/.config/i3/config 2>/dev/null || \
  cp /usr/share/doc/i3/config ~/.config/i3/config 2>/dev/null || \
  i3-config-wizard
```

الإضافات اللي تحطها في `~/.config/i3/config`:

```
### Startup ###
exec --no-startup-id dbus-update-activation-environment --all

exec --no-startup-id pipewire &
exec --no-startup-id dunst &
exec --no-startup-id picom -b &
exec --no-startup-id feh --bg-fill ~/wallpaper.jpg &
exec --no-startup-id nm-applet &
exec --no-startup-id blueman-applet &
exec --no-startup-id udiskie --tray &
exec --no-startup-id /usr/lib/polkit-gnome/polkit-gnome-authentication-agent-1 &
exec --no-startup-id xss-lock -- i3lock -f -c 000000 &

### الـ Bar ###
bar {
    status_command i3status
    # أو لو ثبّتت polybar:
    # exec_always --no-startup-id polybar
}

### Screenshot ###
bindsym Print exec flameshot gui
bindsym Shift+Print exec flameshot full -p ~/Pictures/

### Volume ###
bindsym XF86AudioRaiseVolume exec pactl set-sink-volume @DEFAULT_SINK@ +5%
bindsym XF86AudioLowerVolume exec pactl set-sink-volume @DEFAULT_SINK@ -5%
bindsym XF86AudioMute exec pactl set-sink-mute @DEFAULT_SINK@ toggle

### Brightness ###
bindsym XF86MonBrightnessUp exec brightnessctl set +10%
bindsym XF86MonBrightnessDown exec brightnessctl set 10%-

### Lock ###
bindsym $mod+l exec i3lock -f -c 000000
```

## مشاركة الكونفيج بين i3 و Sway (الطريقة الأذكى)

الكونفيجان متطابقان تقريباً — تقدر تعمل كونفيج مشترك واحد ملفات إضافية منفصلة لكل WM.

### الهيكل

```
~/.config/i3/config          ← الكونفيج المشترك (الـ keybindings، الـ layouts، إلخ)
~/.config/i3/conf.d/i3.conf  ← حاجات خاصة بـ i3 فقط (polybar، picom، i3lock)
~/.config/sway/config        ← symlink للكونفيج المشترك
~/.config/sway/conf.d/sway.conf ← حاجات خاصة بـ Sway فقط (waybar، swaylock)
```

### التنفيذ

```bash
# اعمل الـ symlink
mkdir -p ~/.config/sway/conf.d ~/.config/i3/conf.d
ln -s ~/.config/i3/config ~/.config/sway/config
```

في `~/.config/i3/config` أضف في الآخر:
```
# تحميل الـ extras الخاصة بكل WM
include conf.d/*.conf
```

في `~/.config/i3/conf.d/i3.conf`:
```
exec --no-startup-id picom -b &
exec --no-startup-id polybar &
exec --no-startup-id feh --bg-fill ~/wallpaper.jpg &
bindsym $mod+l exec i3lock -f -c 000000
```

في `~/.config/sway/conf.d/sway.conf`:
```
exec waybar &
exec swaybg -i ~/wallpaper.jpg -m fill &
bindsym $mod+l exec swaylock -f -c 000000
output * bg ~/wallpaper.jpg fill
```

## الـ Theming في i3

i3 بيستخدم **lxappearance** بدل `nwg-look`:

```bash
lxappearance
```

نفس الفكرة — بتختار GTK theme + Icons + Cursor وبتضغط Apply. لكن على X11 بيشتغل أبسط بدون تعقيدات الـ gsettings.

## picom (Compositor لـ i3)

بدونه مش هيبقى عندك transparency أو تأثيرات. أبسط إعداد:

```bash
# شغّله من كونفيج i3
exec --no-startup-id picom -b

# أو بكونفيج خاص
mkdir -p ~/.config/picom
nano ~/.config/picom/picom.conf
```

```ini
# picom.conf — إعداد بسيط
backend = "glx";
vsync = true;
shadow = false;
fading = true;
fade-in-step = 0.05;
fade-out-step = 0.05;

# Transparency للـ terminal
opacity-rule = [
    "90:class_g = 'tilix'",
    "90:class_g = 'Alacritty'"
];
```
